# TIL — 자료구조 기초 (List / LinkedList / Stack / Queue)

**날짜: 2025.12.29 월요일**

## 자료구조란?
프로그램에서 데이터를 **어떻게 저장하고, 배치하고, 사용하는지**에 따라 성능과 효율이 크게 달라진다.
이때 데이터를 효율적으로 관리하기 위해 고안된 저장 방식들을 **자료구조**라고 한다.

자료구조를 적절하게 선택하면:

- 탐색이 빨라지고
- 메모리 사용이 줄어들며
- 프로그램 구조가 더 깔끔해진다.

## List와 LinkedList
### `List<T>` — 동적 배열
`List<T>`는 배열 기반의 순차 자료구조이지만, 크기가 자동으로 늘어나는 **동적 배열**이다.

특징:

- 인덱스로 빠르게 접근 가능 (`list[i]`)
- 중간 삽입/삭제 시 뒤의 요소들을 전부 밀어야 하므로 비용이 큼
- 순차 접근, 정렬, 검색에 유리

주요 기능:

- `Add()` : 뒤에 추가
- `Insert()` : 중간 삽입
- `Remove()`, `RemoveAt()` : 삭제
- `Contains()` : 탐색
- `Sort()` : 정렬

### `LinkedList<T>` — 연결 리스트
`LinkedList<T>`는 각 데이터가 노드(Node)로 연결된 구조로,  
메모리 상에 연속적으로 저장되지 않는다.

특징:

- 노드끼리 연결되어 있어 **삽입/삭제가 빠름**
- 인덱스 접근 불가 (순차 탐색만 가능)
- 중간 삽입/삭제가 잦은 경우에 유리

## Stack과 Queue
### Stack — 후입선출 (LIFO)
Stack은 탑처럼 쌓는 구조로,  
**가장 마지막에 들어간 데이터가 가장 먼저 나오는 구조**이다.

| 기능 | 설명 |
|---|---|
| Push | 데이터 추가 |
| Pop | 맨 위 데이터 꺼내기 |
| Peek | 맨 위 데이터 보기만 하기 |

주로 사용되는 곳:
- 되돌리기(Undo)
- 콜스택
- 게임 스킬 취소
- 카드 덱 구조

### Queue — 선입선출 (FIFO)
Queue는 대기줄처럼  
**먼저 들어온 데이터가 먼저 나오는 구조**이다.

| 기능 | 설명 |
|---|---|
| Enqueue | 데이터 추가 |
| Dequeue | 데이터 꺼내기 |

주로 사용되는 곳:
- 작업 대기열
- 프린터 대기열
- 네트워크 패킷 처리
- 게임 유닛 생산 대기줄

## 오늘의 핵심 정리
| 자료구조 | 특징 |
|---|---|
| List | 빠른 인덱스 접근, 중간 수정은 느림 |
| LinkedList | 삽입/삭제 빠름, 탐색 느림 |
| Stack | 후입선출(LIFO) |
| Queue | 선입선출(FIFO) |

## 느낀 점

자료구조는 단순히 “저장소”가 아니라,  
**프로그램의 규칙과 흐름을 강제하는 구조**라는 점이 인상 깊었다.  
특히 Stack과 Queue를 이용해 실제 게임 시스템(유닛 생산, 카드 덱, 자판기 등)을 모델링할 수 있다는 점이 이해에 도움이 되었다.
