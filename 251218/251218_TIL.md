## Git과 GitHub 개념

- Git: 파일과 변경 이력을 기록하고 관리하는 로컬 버전 관리 도구
- GitHub: Git 기반 원격 저장소를 웹으로 제공하는 서비스
- CLI를 통해 명령어 입력으로 사용

## Git 주요 명령어 및 사용 예시

| 명령어 | 설명	| 사용 예시
|---|---|---|
`git clone`	| 원격 저장소 전체 복사, 커밋 이력 포함	| `git clone https://github.com/user/repo.git`
`git commit` |	로컬 변경 사항 기록 (원격 반영 X) | `git add .`<br>`git commit -m "Add new feature"`
`git push` | 로컬 커밋을 원격 저장소로 업로드 | `git push origin main`
`git checkout` | 특정 커밋이나 브랜치로 이동 | `git checkout feature/login`
`git fetch` | 원격 변경 사항 확인 | `git fetch`
`git pull` | 원격 변경 사항 가져오기 | `git pull origin main`
`.gitignore` | 추적 제외, 민감 정보 보호 | API 키, 비밀번호, 빌드 결과 등
`git reset` | 기록 삭제, 로컬 히스토리 제거 | 사용 시 주의!
`git revert` | 이전 상태로 되돌리고 새로운 커밋 추가 | 안전하게 롤백 시 사용
`branch / merge` | 브랜치 생성 / 합치기 | `git branch feature/ui`<br>`git merge feature/ui`
`main / origin/main` | 로컬 / 원격 기준	팀 협업 시 | 충돌 방지 중요


## Git 흐름 예시 (팀 협업)

- origin/main은 원격 저장소 기준, 로컬 main과 다를 수 있음
- 팀원 B가 pull로 업데이트 후 merge하면 최신 상태 유지
```
원본 저장소(GitHub)
    │
    │ clone
    ▼
개인 로컬 저장소 (A팀원)
    │
    ├─ feature/login 브랜치 생성
    │
    ├─ 작업 후 commit
    │
    └─ push origin feature/login

팀원 B                  팀원 A
git pull origin main     git push origin main
   │                        │
   └─ 변경 사항 통합 ────────┘
```

## 느낀 점 / Tip

- 커밋 단위는 작고 의미 있게

- reset/revert는 위험하므로 사전 백업 필수

- origin 개념 이해가 팀 협업과 충돌 방지 핵심

- .gitignore 잘 설정해두면 불필요한 파일 추적 방지

- clone → branch → commit → push → pull → merge 흐름을 이해하면 협업이 훨씬 편리
