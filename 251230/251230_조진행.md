# 과제 1. 해시함수 나눗셈법 조사
- 키를 테이블 크기로 나눈 나머지를 인덱스로 사용하는 해싱 방법입니다.
```csharp
hash = key % tableSize
```
| Key | 계산      | 인덱스 |
| --- | ------- | --- |
| 23  | 23 % 10 | 3   |
| 47  | 47 % 10 | 7   |
| 91  | 91 % 10 | 1   |

- 사용하는 이유는 계산이 빠르고, 구현이 간단해서 입니다. 그러한 이유로 가장 기본적인 해시 함수입니다.
- 단점 : 테이블 크기 선택이 나쁘면 충돌이 몰립니다.
  - 그래서 테이블 크기를 소수(Prime)으로 사용합니다.


# 과제 2. 해시 충돌 해결방안 - 개방주소법(선형탐사) 조사
- 해시 과정에서 충돌이 발생할 경우, 해당 Key값의 비어있는 다음 위치에 값을 넣습니다.  
비어있는 자리를 차례대로 탐색해나가 선형 조사법이라고 합니다.
- 충돌이 적게 발생한다면 매우 빠른 데이터 접근 속도를 가집니다.

- 단점
  - 충돌이 발생하기 시작하면, 유사한 값을 가지는 데이터끼리 밀집되는 '집중 결합' 문제가 존재합니다.
  - 해시 테이블 크기를 크게 만들면 됩니다. 단, 테이블 크기가 크면 메모리 소모량 또한 높아집니다.
