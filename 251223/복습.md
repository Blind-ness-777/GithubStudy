# 오늘 수업 내용, 사실 이렇게 묶인다 (핵심 정리)
- 오늘 배운 걸 개념 단위로 쪼개면 6개야.

## 1️⃣ 참조 타입 vs 값 타입 (배열/클래스 핵심)
- 배열, 클래스 → 참조 타입
- 함수로 전달되면 주소가 복사됨
- 그래서:
  - 배열 요소 바꾸면 원본 영향 ⭕
  - 배열 자체를 새로 할당하면 원본 영향 ❌

**👉 이게 문제 2번 / 3번 차이의 본질**

## 2️⃣ new의 정확한 의미
- new = 힙에 공간 만들고
-  그 주소를 참조 변수에 넣는다

절차지향이든 객체지향이든:
- new int[3]
- new Monster()

**👉 다 똑같은 개념**

## 3️⃣ static
- 프로그램 실행 중 딱 하나
- 인스턴스랑 무관
- 값 하나 + 주소 하나

그래서:

- “같은 값” ❌
- “같은 메모리 주소” ⭕

## 4️⃣ 상속
- 공통 필드/메서드를 부모로 뽑아냄
- 자식은:
  - 그대로 쓰거나
  - override로 바꾸거나

**👉 코드 줄이기 + 유지보수 목적**

## 5️⃣ 오버로딩 / 오버라이딩
- 오버로딩:  
  **👉 같은 이름, 다른 매개변수**
- 오버라이딩:  
  **👉 부모 virtual → 자식 override**

## 6️⃣ 업캐스팅 / 다운캐스팅 (is / as)
- 업캐스팅: 항상 안전
- 다운캐스팅: 실제 객체 타입일 때만 가능
- 기준은 항상:
  - new로 뭘 만들었냐

## ❗ 지금 복습에서 가장 중요한 것 (이거 하나면 됨)

>“이게 왜 필요한지”까진 이해 안 돼도 된다  
>“이게 언제 쓰이는지만 알면 성공”

예:
- static → 전역 상태
- 상속 → 같은 종류 묶기
- override → 행동 다르게 하기
- is/as → 진짜 자식인지 확인
